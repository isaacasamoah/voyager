## How It Works

**1. You bring the start state**
- A rough idea
- An uploaded document
- A goal to explore

**2. Shipwright refines with you**
- Asks clarifying questions
- Makes specific suggestions
- Shows changes in real-time
- Explains WHY when it helps you learn

**3. You decide when it's ready**
- Messages: Ready to send
- Documents: Ready to submit
- Whiteboards: Idea is clear (5-year-old test)

## What Makes Shipwright Different

### vs Navigator (Questioning)
**Navigator** helps you think through problems by asking questions.
**Shipwright** helps you create artifacts by editing together.

### vs Cartographer (Conversation)
**Cartographer** extracts your knowledge through interview.
**Shipwright** refines your work through collaborative editing.

---

## Tips for Great Sessions

**Start with something**
Don't ask Shipwright to write from scratch. Bring a rough draft, an idea, or an uploaded document.

**Make decisions**
When Shipwright suggests changes, say yes/no/maybe. You're in control.

**Ask "why" when it helps**
"Why did you phrase it this way?" â†’ Learn transferable principles

**Iterate quickly**
Short back-and-forth beats long explanations. Doing teaches faster than hearing.

**Know when you're done**
- Messages: Would you send this?
- Documents: Would you submit this?
- Whiteboards: Can you explain this simply?
