# Learning Principles - Claude Code Integration

## 🎯 **Core Principle**

> **"Real skill requires structured progression, guided practice, and most importantly, immediate feedback when you're doing something wrong."**

This applies to every coding session in this repository.

---

## 🔄 **The Code Review Cycle**

### **When Working on New Features:**

```
1. UNDERSTAND: Read requirements, ask questions
2. PLAN: Outline approach before coding
3. IMPLEMENT: Write code incrementally
4. TEST: Run and verify each piece
5. REVIEW: Get feedback immediately
6. ITERATE: Fix issues while context fresh
7. INTEGRATE: Ensure fits with existing code
```

---

## ✅ **Before Committing Code**

**Quality Checklist:**
- [ ] Code works correctly
- [ ] Follows Python conventions (PEP 8)
- [ ] Error handling present
- [ ] Type hints included
- [ ] Docstrings written
- [ ] Tests pass
- [ ] Integrates cleanly with existing code

**Understanding Check:**
- [ ] Can explain what this code does
- [ ] Can explain why this approach was chosen
- [ ] Know what trade-offs were made

---

## 🚀 **Session Pattern**

### **Starting a Feature:**
1. Understand requirements clearly
2. Ask questions if anything unclear
3. Outline approach before coding
4. Get feedback on plan before implementing

### **During Implementation:**
1. Code incrementally (small pieces)
2. Test each piece as you go
3. Request review when chunk is complete
4. Don't wait until everything is "done"

### **Stuck on Something:**
1. Try debugging for 10-15 minutes
2. If still stuck, ask for help immediately
3. Don't waste time on trial-and-error
4. Learn the correct approach

---

## 💡 **Learning from Feedback**

**When receiving code review feedback:**
- ✅ Read carefully and understand the issue
- ✅ Ask clarifying questions if needed
- ✅ Fix immediately while context fresh
- ✅ Learn the pattern to apply elsewhere
- ✅ Thank reviewer for catching issues

**When feedback reveals gaps:**
- ✅ Ask for explanation of concept
- ✅ Request examples or references
- ✅ Practice the correct pattern
- ✅ Apply to future code

---

## 🎯 **Skill Development Goals**

**Short-term (Per Session):**
- Implement features correctly
- Follow established patterns
- Write clean, readable code
- Get faster at common tasks

**Long-term (Project-wide):**
- Build production-quality instincts
- Recognize patterns across problems
- Propose good solutions independently
- Catch own mistakes before review

---

## 📚 **Quick Reference**

**For every new concept:**
1. Make sure you understand it
2. Implement a simple example
3. Get feedback immediately
4. Fix and iterate
5. Apply to real project code

**For every bug:**
1. Reproduce reliably
2. Understand root cause
3. Fix properly (not just make it work)
4. Learn the pattern to avoid future

**For every feature:**
1. Plan before coding
2. Implement incrementally
3. Test continuously
4. Review before considering done
5. Integrate cleanly

---

**Remember: The goal is learning, not just completing features. Building skills correctly from the start prevents compounding mistakes later.**

🚀 *Better to move slowly and correctly than quickly and wrong.*
